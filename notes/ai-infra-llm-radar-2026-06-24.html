3 SGLang + Dynamo 在 DeepSeek-V4/GB300 上的后 Day-0 优化 PyTorch blog: June 2026 MTP curve 在 GB300 disaggregated lane 达到约 11,200 tok/s/GPU,对比 April Day-0 no-MTP 约 2,200 tok/s/GPU,同等交互性下约 5x。 前沿模型服务的难点已从“能不能跑”转成“kernel、runtime、CUDA graph、MoE、compression、disaggregation 能否一起稳定”。

3. 趋势分析: 历史、当下、未来

3.1 历史: 从“模型可服务化”到“推理栈分层”

2022-2024 的主线是把 Transformer 高效变成 API:FlashAttention、PagedAttention、continuous batching、tensor parallel、streaming、OpenAI-compatible endpoint。TGI、vLLM、TensorRT-LLM、llama.cpp、Ollama 分别从生产服务、吞吐、硬件优化、本地可移植和开发体验切入。那一阶段的核心指标是 tokens/sec、TTFT、显存占用、batch size、模型兼容性。

3.2 当下: Agentic workload 让瓶颈沿链路迁移

当下的核心变化是一次任务不再只是单次 chat completion。Agent 会读文件、检索文档、调用工具、运行测试、写 PR、触发 CI、等待 review,再继续迭代。于是成本和可靠性瓶颈沿着“上下文 → prefill/KV → decode/kernel → router/control plane → tools/agents → platform/governance”移动。

  • 证据 1: GitHub Agentic Workflows 把 agent 放进 Actions,同时强调 firewall、safe outputs、threat detection,说明平台层的安全边界成为必要条件。
  • 证据 2: LMCache v0.5.0 的 P2P transfer 和 llm-d v0.8.0 的 multi-tier KV offloading 都把 KV cache 明确推向分布式资源。
  • 证据 3: SGLang DeepSeek-V4 文章显示前沿 serving 的提升来自 kernel、runtime、CUDA graph、speculative path、Dynamo 协同 bugfix,不是单点优化。
  • 证据 4: Ray Serve LLM 文章说明 orchestration overhead 已成为可量化性能瓶颈,并通过 direct streaming / HAProxy / vLLM executor 下降。

3.3 未来: AI Infra 会变成“分层 agent 操作系统”

未来 6-18 个月的趋势不是某一个 engine 通吃,而是栈分层更清楚:模型层选择能力和成本,推理层压吞吐和延迟,缓存层减少重复 prefill,协议层连接工具和数据,调度层管理 GPU 与 SLO,治理层处理权限、审计和复现。

  1. 模型层: frontier model 承接复杂推理,open-weight / local model 承接低延迟、隐私、成本敏感任务。
  2. 推理层: vLLM / SGLang / TensorRT-LLM / llama.cpp 分别服务云端高吞吐、复杂生成、NVIDIA 极致优化和边缘/本地可移植。
  3. 缓存层: KV cache 变成可配额、可压缩、可迁移、可观测的共享资源,并从 HBM 扩展到 CPU、NVMe 和远端存储。
  4. 协议层: MCP 这类工具协议会像 HTTP/gRPC 之于微服务一样,成为 agent 接入外部世界的标准边界。
  5. 调度层: Gateway API Inference Extension、llm-d、Kueue、KServe、Ray 会把 LLM 流量特征带进 Kubernetes 和通用分布式运行时。
  6. 治理层: prompt/response hash、tool-call audit、PII 处理、sandbox 隔离、policy enforcement 和 safe outputs 会成为企业默认项。

4. 一句话结论

2026-06-24 的 AI Infra 关键词是 distributed state:模型能力仍重要,但更稀缺的是能把 KV cache、prefill/decode、router、agent tools、CI 平台和安全治理串成稳定闭环的系统能力。过去拼单次调用,现在拼可复用上下文;未来拼的是可恢复、可审计、可跨云调度的 agent 操作层。
