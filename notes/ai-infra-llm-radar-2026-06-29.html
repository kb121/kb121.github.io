3. 趋势分析: 历史、当下、未来

3.1 历史: 从“模型服务化”到“agent 操作层”

2022-2023 的核心是把 Transformer 变成可用 API:FlashAttention、PagedAttention、连续批处理、tensor parallel、token streaming、OpenAI-compatible endpoint。2024-2025 的重心转向复杂生成、prefix cache、MCP、agent state、Kubernetes inference routing。到 2026,系统问题进一步扩展为 speculative decoding、multi-tier KV、P/D disaggregation、gateway security、tool-call audit 和可恢复执行。

3.2 当下: 瓶颈从单点吞吐迁移到端到端链路

当下的 agent 任务不再只是一次 chat completion。它会读代码、检索文档、调用工具、运行测试、写 PR、触发 CI、等待 review,再继续迭代。因此成本和可靠性瓶颈沿着“上下文 → prefill/KV → speculative decode → router → tools/session → gateway/security → 平台容量”移动。

  • 证据 1: DeepSpec / DSpark 将 speculative decoding 的训练与评估开源,公开报道给出 60%-85% per-user generation speedup,说明 decoding 调度已是生产竞争点。
  • 证据 2: SGLang v0.5.14 同时更新 GB300、MoE dispatch、linear-attention cache 和 Blackwell prefill kernel,说明 serving engine 正贴近硬件与模型结构共同优化。
  • 证据 3: vLLM v0.23.0 的 multi-tier KV offloading 与 LMCache operator 说明 KV 已从 engine 内部状态变成平台资源。
  • 证据 4: LiteLLM v1.90.0 强调 cosign image signature,说明 AI gateway 的供应链可信性正在成为企业部署默认要求。

3.3 未来: AI Infra 会变成“分层 agent 操作系统”

未来的竞争点不是某个 engine 通吃,而是栈分层更清楚:模型层选择能力和成本,推理层压吞吐和延迟,缓存层减少重复 prefill,协议层连接工具和数据,调度层管理 GPU 与 SLO,治理层处理权限、审计和复现。6/29 的新增证据让这个判断更强:DSpark 把“decode 几个 token”变成动态调度问题,LiteLLM 把 gateway 镜像签名显式化,llm-d 把 serving 组件拆成 router/EPP/sidecar/KV cache。

  1. 模型层: frontier model 承接复杂推理,open-weight / local model 承接低延迟、隐私、成本敏感任务。
  2. 推理层: vLLM / SGLang / TensorRT-LLM / llama.cpp 分别服务云端高吞吐、复杂生成、NVIDIA 极致优化和边缘/本地可移植。
  3. 缓存层: KV cache 变成可配额、可压缩、可迁移、可观测的共享资源,命中率会进入 router 决策。
  4. 协议层: MCP 这类工具协议会像 HTTP/gRPC 之于微服务一样,成为 agent 接入外部世界的标准边界。
  5. 调度层: Gateway API Inference Extension、llm-d、Kueue 会把 LLM 流量特征带进 Kubernetes 调度。
  6. 治理层: image signing、tool-call audit、PII 处理、sandbox 隔离和 policy enforcement 会成为企业默认项。

4. 一句话结论

2026-06-29 的 AI Infra 关键词是 dynamic serving control:同一模型的成本差距越来越来自动态 speculative decoding、KV cache 命中、模型感知路由、低精度硬件、agent session 可靠性和 gateway 可信供应链。过去拼模型调用,现在拼系统闭环;未来拼的是能否把模型能力稳定、便宜、可恢复、可审计地接入真实工作流。
