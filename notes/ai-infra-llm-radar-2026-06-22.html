3. 趋势分析: 历史、当下、未来

3.1 历史: 从“能跑模型”到“能服务模型”

2022-2024 的 AI Infra 主线是把 Transformer 变成 API:FlashAttention、PagedAttention、连续批处理、tensor parallel、token streaming、OpenAI-compatible endpoint。TGI、vLLM、TensorRT-LLM、llama.cpp、Ollama 分别从生产服务、吞吐、硬件优化、本地可移植和开发体验切入。这个阶段的核心指标是 tokens/sec、TTFT、显存占用、batch size、模型兼容性。

3.2 当下: Agentic workload 让瓶颈沿链路迁移

当下的核心变化是一次任务不再只是单次 chat completion。Agent 会读文件、检索文档、调用工具、运行测试、写 PR、触发 CI、等待 review,再继续迭代。于是成本和可靠性瓶颈沿着“上下文 → prefill/KV → decode/kernel → router/scheduler → 工具协议 → 平台容量 → 审计治理”移动。

  • 证据 1: GitHub 5 月可用性报告写明流量快速增长主要由 AI-assisted / agentic development workflows 驱动,并披露 4 个月内有效容量翻倍、Azure monolith traffic 达 40%、repository replication 达 99%。
  • 证据 2: Red Hat 把 LLM inference 拆成 prefill 与 decode 两个 workload,分别影响 TTFT 与 TPOT,并把 KV cache hit rate 列为核心 KPI。
  • 证据 3: IBM Research 的 llm-d 实验说明 prefix-cache-aware routing 在异构 GPU 池里可把吞吐从约 9,600 tokens/s 提到 14,200 tokens/s。
  • 证据 4: Anthropic Sonnet 5、Google ADK Go 2.0、OpenAI Agents SDK 都把 tool use、sandbox、resume、HITL、trace 作为 agent 工程化能力,说明 infra 需求已经上移到执行层。

3.3 未来: AI Infra 会变成“分层 agent 操作系统”

未来 6-18 个月的趋势不是某一个 engine 通吃,而是栈分层更清楚:模型层选择能力和成本,推理层压吞吐和延迟,缓存层减少重复 prefill,协议层连接工具和数据,调度层管理 GPU/CPU 与 SLO,治理层处理权限、审计和复现。

  1. 模型层: frontier model 承接复杂推理,Sonnet-class / open-weight / local model 承接成本敏感、低延迟和隐私场景。
  2. 推理层: vLLM / SGLang / TensorRT-LLM / llama.cpp 分别服务云端高吞吐、复杂生成、NVIDIA 极致优化和边缘/本地可移植。
  3. 缓存层: KV cache 变成可配额、可压缩、可迁移、可观测的共享资源,并与 router、RDMA/NIXL、prefill/decode pool 绑定。
  4. 协议层: MCP、skills、AGENTS.md、safe outputs、tool allowlist 会像 HTTP/gRPC 之于微服务一样,成为 agent 接入外部世界的标准边界。
  5. 调度层: Gateway API Inference Extension、llm-d、Kueue 会把 LLM 流量特征带进 Kubernetes 调度;异构 GPU/CPU/专用芯片将按 workload phase 分工。
  6. 治理层: prompt/response hash、tool-call audit、PII 处理、sandbox 隔离、threat detection 和 policy enforcement 会成为企业默认项。

4. 一句话结论

2026-06-30 的 AI Infra 关键词是 agentic operating pressure:更强、更便宜的 agent 模型把任务量推高;KV cache、低精度 kernel、disaggregated serving、Kubernetes-aware routing、sandbox/resume 和治理能力共同决定真实成本。过去拼模型调用,现在拼系统闭环;未来拼的是能否把模型能力稳定、便宜、可恢复、可审计地接入真实工作流。
