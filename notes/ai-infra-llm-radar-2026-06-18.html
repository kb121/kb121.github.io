3. 技术趋势分析: 历史、当下、未来

3.1 历史: 从“能跑模型”到“服务模型”

2022-2024 的主线是把 Transformer 模型部署成 API:张量并行、连续批处理、PagedAttention、token streaming、OpenAI-compatible endpoint。TGI、vLLM、TensorRT-LLM、llama.cpp、Ollama 分别从生产服务、吞吐、硬件优化、本地推理和开发体验切入。这个阶段的核心指标是 tokens/sec、TTFT、显存占用、batch size。

3.2 当下: Agentic workload 让瓶颈迁移

现在的瓶颈不再只是一次 chat completion 的吞吐。Agent 会反复读代码、检索文档、调用工具、运行测试、写 PR、触发 CI。于是系统瓶颈沿着这条链路迁移:

  • 证据 1: vLLM v0.23.0 release notes 将 multi-tier KV cache offloading、MRv2、Rust frontend、DeepSeek-V4 hardening 放在核心更新里。
  • 证据 2: SGLang v0.5.13 将 Spec V2、Breakable CUDA Graph、HiCache 和 DeepSeek V4 sparse attention 合并进 serving 路线。
  • 证据 3: Dynamo 文档把 disaggregated serving、KV-aware routing、SLA planner 定义为 vLLM/SGLang/TensorRT-LLM 之上的编排能力。
  • 证据 4: GitHub 官方报告说明 agentic development 流量已经影响 monolith、数据库隔离、认证、repo replication 和云容量迁移。

3.3 未来: AI Infra 会变成“分层 agent 操作系统”

未来 6-18 个月的方向,大概率不是某个单点 engine 获胜,而是出现更清晰的分层:

  1. 模型层: 闭源 frontier model 做复杂推理,开源/本地模型承接成本敏感、隐私敏感、低延迟任务。
  2. 推理层: vLLM / SGLang / TensorRT-LLM / llama.cpp 分别服务云端高吞吐、结构化生成、NVIDIA 极致优化和本地边缘。
  3. 缓存层: KV cache 从 engine 内部状态变成跨请求、跨节点、跨 engine 的资源调度问题。
  4. 协议层: MCP 这类工具协议会像 HTTP/gRPC 之于微服务一样,成为 agent 连接外部世界的契约。
  5. 治理层: audit log、model version、prompt/response hash、policy、PII 处理会成为企业部署默认要求。

4. 一句话结论

2026 年的 AI Infra 关键词是 agentic pressure: agent 把推理成本、上下文复用、低精度硬件、工具协议、平台容量和合规审计同时推到前台。过去拼模型调用,现在拼系统闭环;未来拼的是能否把模型能力稳定、便宜、可审计地接入真实工作流。
