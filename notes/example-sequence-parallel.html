DeepSpeed-Ulysses 序列并行入门A Primer on DeepSpeed-Ulysses Sequence Parallelism

训练长序列模型时,激活值显存随序列长度线性增长,注意力部分甚至接近平方级。当序列推到 32K、64K,单卡显存很快就不够用了。这篇笔记记录我在 Wan2.1 14B 文生视频模型上做长序列训练时,为什么选了 DeepSpeed-Ulysses 序列并行,以及它的核心思想。 When training long-sequence models, activation memory grows linearly with sequence length — and the attention block grows closer to quadratically. Push the sequence to 32K or 64K and a single device runs out of memory fast. These are my notes on why I picked DeepSpeed-Ulysses sequence parallelism for long-sequence training of the Wan2.1 14B text-to-video model, and how it works.

问题:显存从哪来The problem: where memory goes

先做一次显存 profiling,把占用拆成三块:参数与优化器状态、激活值、临时缓冲。长序列场景下激活值是绝对大头,而它正比于 batch × seq_len × hidden。要扩 seq_len,要么砍 batch,要么把序列这一维切到多卡上。 Start with a memory profile and split usage into three buckets: parameters + optimizer state, activations, and scratch buffers. For long sequences, activations dominate, and they scale with batch × seq_len × hidden. To grow seq_len, you either shrink the batch or shard the sequence dimension across devices.

Ulysses 的核心:沿序列维切分 + all-to-allThe core idea: shard the sequence, then all-to-all

Ulysses 的做法是:大部分时间每张卡只持有完整序列的一个分片(沿 seq 维切);到了注意力计算这一步,用一次 all-to-all 把数据重排成"沿 head 维切",这样每张卡拿到完整序列、但只算自己负责的几个 attention head;算完再 all-to-all 切回去。 Ulysses keeps each device holding only a shard of the full sequence (split along the seq dimension) most of the time. At the attention step, a single all-to-all reshuffles the data to be split along the head dimension instead — so each device now sees the full sequence but computes only its assigned attention heads. A second all-to-all switches the layout back afterward.

直觉:注意力需要"完整序列",其它算子需要"完整 hidden"。Ulysses 用两次 all-to-all 在这两种布局之间来回切换,从而两边都不爆显存。 Intuition: attention needs the full sequence; everything else needs the full hidden dim. Ulysses uses two all-to-alls to flip between those two layouts so neither blows up memory.

结果Results

在 Wan2.1 14B 上,32K 序列的训练从 32 卡降到了 8 卡,节点数下降 75%、整体显存占用下降 50%,并且在相同卡数下能支持更长的序列。 On Wan2.1 14B, 32K-sequence training went from 32 GPUs down to 8 — a 75% cut in nodes and a 50% drop in overall memory, while also enabling longer sequences at the same device count.

什么时候用When to reach for it

  • 序列很长、激活值是瓶颈时,序列并行比单纯加 ZeRO 更直接。When sequences are long and activations are the bottleneck, sequence parallelism is more direct than just stacking ZeRO.
  • 可以和数据并行 / ZeRO 组合,形成多维并行。It composes with data parallelism / ZeRO into multi-dimensional parallelism.
  • 注意 all-to-all 的通信开销,在带宽不足的互联上收益会打折。Watch the all-to-all communication cost — the win shrinks on bandwidth-starved interconnects.
