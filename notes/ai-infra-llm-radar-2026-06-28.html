3. 趋势分析: 历史、当下、未来

3.1 历史: 从“能跑模型”到“能服务模型”

2022-2024 的 AI Infra 主线是把 Transformer 变成稳定 API:FlashAttention、PagedAttention、连续批处理、tensor parallel、token streaming、OpenAI-compatible endpoint。TGI、vLLM、TensorRT-LLM、llama.cpp、Ollama 分别从生产服务、吞吐、硬件优化、本地可移植和开发体验切入。这个阶段的核心指标是 tokens/sec、TTFT、显存占用、batch size、模型兼容性。

3.2 当下: Agentic workload 让瓶颈沿链路迁移

当下的核心变化是一次任务不再只是单次 chat completion。Agent 会读文件、检索文档、调用工具、运行测试、写 PR、触发 CI、等待 review,再继续迭代。于是成本和可靠性瓶颈沿着“上下文 → prefill/KV → decode/kernel → router → 工具协议 → 平台容量 → 审计治理”移动。

  • 证据 1: GitHub 5 月可用性报告直接把流量增长归因于 AI-assisted / agentic workflows,并说明正在迁移 Azure、拆分 monolith、隔离认证授权等共享失败点。
  • 证据 2: SGLang 0.5.14 把 DeepSeek-V4、MoE load balancing、Blackwell prefill kernel 和 prefix-cache memory savings 放在 release highlights,说明复杂模型 serving 已经进入 kernel / routing / cache 一体优化阶段。
  • 证据 3: Dynamo 1.0 文档把 NIXL、KVBM、Flash Indexer、multi-tier storage 与 agentic subagent cache reuse 连接起来,说明 KV cache 正被提升为跨节点基础设施。
  • 证据 4: llm-d router v0.9.0 加入 prefix-cache estimation、TTFT/ITL metrics 并降低大规模 fleet 调度内存分配,说明 Kubernetes inference router 正从实验走向生产。

3.3 未来: AI Infra 会变成“分层 agent 操作系统”

未来 6-18 个月的趋势不是某一个 engine 通吃,而是栈分层更清楚:模型层选择能力和成本,推理层压吞吐和延迟,缓存层减少重复 prefill,协议层连接工具和数据,调度层管理 GPU 与 SLO,治理层处理权限、审计和复现。

  1. 模型层: frontier model 承接复杂推理,open-weight / local model 承接低延迟、隐私、成本敏感任务。
  2. 推理层: vLLM / SGLang / TensorRT-LLM / llama.cpp 分别服务云端高吞吐、复杂生成、NVIDIA 极致优化和边缘/本地可移植。
  3. 缓存层: KV cache 变成可配额、可压缩、可迁移、可观测、可设置 TTL 的共享资源。
  4. 协议层: MCP 这类工具协议会像 HTTP/gRPC 之于微服务一样,成为 agent 接入外部世界的标准边界。
  5. 调度层: Gateway API Inference Extension、llm-d、Kueue 会把 LLM 流量特征带进 Kubernetes 调度。
  6. 治理层: prompt/response hash、tool-call audit、PII 处理、sandbox 隔离和 policy enforcement 会成为企业默认项。

4. 一句话结论

2026 年中期的 AI Infra 关键词是 agentic pressure:agent 把推理成本、上下文复用、低精度硬件、工具协议、Kubernetes 调度、开发平台容量和合规审计同时推到前台。过去拼模型调用,现在拼系统闭环;未来拼的是能否把模型能力稳定、便宜、可恢复、可审计地接入真实工作流。
