趋势分析(带证据)Trend analysis (with evidence)

历史:从「模型 / 框架」到「框架 / 应用」History: from "models / frameworks" to "frameworks / apps"

回看 GitHub trending 的主角变迁:2022-11 ChatGPT 引爆后,2023-03 的 Auto-GPT 是第一个病毒式的「autonomous agent」,数周内冲到十万星;紧随其后是 prompt 合集、LangChain / LlamaIndex 这类编排框架,以及 llama.cpp、Ollama 带来的本地推理浪潮。证据是这些仓库当年都长期霸榜——主角是「模型权重 + 编排框架」。 Looking back at who led GitHub trending: after ChatGPT (2022-11), Auto-GPT (2023-03) was the first viral "autonomous agent," hitting 100k stars in weeks; then came prompt collections, orchestration frameworks like LangChain / LlamaIndex, and the local-inference wave from llama.cpp and Ollama. The evidence is that these repos dominated the charts for months — the protagonists were "model weights + orchestration frameworks."

当下:分发单元变成了「agent 技能」+ 效率基建Present: the unit of distribution is now the "agent skill" + efficiency infra

今天榜单本身就是证据:当日 star 前四名里有三个是 *-skill(taste-skill、last30days-skill,以及 OSSInsight 24h 榜上的 graphify),指向 2025 年下半年兴起的 Agent Skills 范式——把能力做成小而可组合的「技能包」,挂载到 Claude / Cursor / Codex 等 agent 上,而不是再造一个大框架。与此同时:(a) 通用/编码 agent 走向开源可扩展(goose 用 Rust 重写、hermes-agent);(b) 检索基建在拼效率——turbovec 用 量化(TurboQuant)压索引,OSSInsight 24h 榜上的 headroom 在压 LLM 输入 token(60–95%),说明「降本」成了新战场;(c) 大厂在围绕 agent 重构底座——openai/plugins 与 microsoft/pg_durable(durable execution 是长跑 agent 的可靠性地基);(d) 本地 / 离线 / 主权信号同时出现——project-nomad(离线 AI 计算机)、ChinaTextbook(主权开放内容)。 Today's board is itself the evidence: three of the top four by stars-today are *-skill repos (taste-skill, last30days-skill, plus graphify on OSSInsight's 24h board), pointing to the Agent Skills paradigm that took off in late 2025 — capabilities packaged as small, composable "skill packs" that mount onto agents (Claude / Cursor / Codex) instead of yet another big framework. In parallel: (a) general/coding agents go open and extensible (goose rewritten in Rust, hermes-agent); (b) retrieval infra competes on efficiency — turbovec compresses indexes via quantization (TurboQuant), and headroom on the 24h board compresses LLM input tokens (60–95%), showing "cost reduction" is the new battleground; (c) incumbents re-platform around agents — openai/plugins and microsoft/pg_durable (durable execution = the reliability substrate for long-running agents); (d) local / offline / sovereignty signals appear together — project-nomad (offline AI computer), ChinaTextbook (sovereign open content).

未来:技能市场 + 持久化 agent + 检索极致降本Future: skill marketplaces + durable agents + ruthlessly cheaper retrieval

从今天的信号外推,三条线最可能延续:(1) 「技能市场」生态化——agent 能力像 npm 包一样被发现、安装、版本化,MCP(Model Context Protocol)等标准会加速这一过程;(2) 持久化 / 有状态 agent——pg_durable 这类「数据库内 durable execution」会成为长跑、可恢复、可审计 agent 工作流的标配;(3) 检索与上下文的效率前沿——量化向量索引(turbovec)+ token 压缩(headroom)把 RAG 的单位成本继续往下打;叠加模型变小,(4) 本地 / 边缘 agentic AI 会更普及。可证伪的预测:未来几周 trending 上 *-skill / MCP 相关仓库占比会维持高位,而纯「prompt 合集」类仓库继续退场。 Extrapolating from today's signals, three lines are most likely to continue: (1) "skill marketplaces" become an ecosystem — agent capabilities get discovered, installed, and versioned like npm packages, accelerated by standards like MCP (Model Context Protocol); (2) durable / stateful agents — "in-database durable execution" like pg_durable becomes table stakes for long-running, resumable, auditable agent workflows; (3) the efficiency frontier in retrieval & context — quantized vector indexes (turbovec) + token compression (headroom) keep driving RAG unit cost down; and with models shrinking, (4) local / edge agentic AI spreads. A falsifiable prediction: over the next few weeks, *-skill / MCP-related repos stay a large share of trending, while pure "prompt-collection" repos keep fading.
