未来:Harness 即产品 + 上下文工程标准化 + 本地记忆默认化Future: the harness is the product + context engineering standardizes + local memory by default

从本期信号外推:(1) Agent harness 即产品 —— openclaw(380.5k)、ECC(221.7k)、superpowers(238.7k)说明人们 star 的是「把模型用好的那套外壳」,模型本身反而退到幕后;(2) 上下文工程标准化 —— design.md、CLAUDE.md(karpathy-skills)、agentskills.io 这类「结构化上下文/技能格式」会像配置文件标准一样固化;(3) 本地代码图谱 + MCP 记忆默认化 —— codegraph / codebase-memory-mcp / Understand-Anything 指向「本地、省 token、随改随同步」的检索记忆成为编码 agent 标配;(4) 沙箱默认化 —— apple/container 这类轻量隔离会成为跑不可信技能的默认运行时;(5) 垂直 agent 应用爆发 —— OpenMontage(视频)、ai-berkshire / daily_stock_analysis(金融)、ai-website-cloner(建站)预示「行业 × 编码 agent」会持续出爆款。可证伪预测:未来几期,trending 头部仍由「harness + 记忆/图谱 + 上下文规范」占据,纯模型/纯框架仓库进不了 Top 10;「代码图谱 / MCP 记忆」类仓库的占比继续上升。Extrapolating: (1) the harness is the product — openclaw (380.5k), ECC (221.7k), superpowers (238.7k) show people star “the shell that makes the model useful,” with the model itself receding; (2) context engineering standardizes — structured context/skill formats like design.md, CLAUDE.md (karpathy-skills), and agentskills.io solidify like config-file standards; (3) local code-graphs + MCP memory by default — codegraph / codebase-memory-mcp / Understand-Anything point to local, token-thrifty, auto-syncing retrieval memory becoming table stakes for coding agents; (4) sandbox by default — lightweight isolation like apple/container becomes the default runtime for untrusted skills; (5) vertical agent apps proliferate — OpenMontage (video), ai-berkshire / daily_stock_analysis (finance), ai-website-cloner (site building) signal more “industry × coding-agent” breakouts.
