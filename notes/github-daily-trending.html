趋势分析(带证据)Trend analysis (with evidence)

历史:从「模型 / 框架」到「框架 / 应用」再到「技能」History: from "models / frameworks" to "frameworks / apps" to "skills"

回看 GitHub trending 的主角变迁:2022-11 ChatGPT 引爆后,2023-03 的 Auto-GPT 是第一个病毒式的「autonomous agent」,数周内冲到十万星;随后是 prompt 合集、LangChain / LlamaIndex 这类编排框架,以及 llama.cpp、Ollama 带来的本地推理浪潮。证据是这些仓库当年长期霸榜——主角是「模型权重 + 编排框架」。到 2024–2025,编码 agent(goose 等)与「应用化」工具接棒;而 2025 下半年起,分发单元进一步细化为可插拔的 agent 技能。 Looking back at who led GitHub trending: after ChatGPT (2022-11), Auto-GPT (2023-03) was the first viral "autonomous agent," hitting 100k stars in weeks; then came prompt collections, orchestration frameworks like LangChain / LlamaIndex, and the local-inference wave from llama.cpp and Ollama — these dominated the charts for months, with "model weights + orchestration frameworks" as protagonists. Through 2024–2025 coding agents (goose et al.) and "productized" tools took over; and from late 2025 the unit of distribution narrowed further into pluggable agent skills.

当下:技能化范式被巨头采纳,且开始长出「协议层」Present: the skills paradigm goes mainstream, and a "protocol layer" appears

今天的榜单给出三条强证据:(1) 技能化已被官方采纳——google/skills(Google 官方 Agent Skills)与昨天的 openai/plugins 一同上榜,加上指南类的 claude-howto 与 PM 技能市场 pm-skills(同日 24h 榜),说明「技能包」不再只是 indie 玩法,而是大厂的官方分发方式;(2) 连续性——last30days-skill 与 turbovec 连续两日分列第一、第二,是「研究型技能 + 量化向量检索」需求真实、非一次性刷榜的证据;(3) 协议层开始出现——CopilotKit 主推 AG-UI 协议(agent 与前端的交互标准),与 MCP(agent 与工具/数据的标准)形成上下两层,Agent-Reach 则把「读全网」做成零成本 CLI 工具。与此同时,roboflow/supervision(CV 感知)与 ChinaTextbook(开放内容)提醒我们:感知层与数据/内容主权这两条副线仍在并行。 Today's board gives three strong pieces of evidence: (1) skills are now official — google/skills (Google's official Agent Skills) charts alongside yesterday's openai/plugins, plus the guide-style claude-howto and the PM skill marketplace pm-skills (same-day 24h board), showing "skill packs" are no longer just an indie move but an incumbent distribution channel; (2) continuity — last30days-skill and turbovec rank No. 1 and No. 2 two days in a row, evidence that demand for "research skills + quantized vector retrieval" is real, not a one-off spike; (3) a protocol layer is emerging — CopilotKit pushes the AG-UI protocol (a standard for agent⇄frontend interaction), forming an upper tier above MCP (the standard for agent⇄tools/data), while Agent-Reach turns "read the whole web" into a zero-cost CLI tool. Meanwhile roboflow/supervision (CV perception) and ChinaTextbook (open content) remind us the perception layer and the data/content-sovereignty side-threads keep running in parallel.

未来:技能市场标准化 + 双协议(MCP × AG-UI)+ 检索极致降本Future: standardized skill marketplaces + dual protocols (MCP × AG-UI) + ruthlessly cheaper retrieval

从今天的信号外推:(1) 技能市场走向标准化——巨头(Google、OpenAI)亲自下场后,agent 技能会像包管理器里的依赖一样被发现、版本化、签名与审计;(2) 双协议成型——下层 MCP(工具/数据)+ 上层 AG-UI(前端/交互)很可能共同定义「agent 应用」的标准栈,CopilotKit 的上榜是早期信号;(3) 检索与上下文持续降本——turbovec 的量化向量索引连续登榜,叠加 token 压缩,把 RAG 单位成本继续往下打;(4) 感知不会消失——supervision 说明 agent 要「看见世界」仍需专门的 CV/多模态工具,未来 agent 工具链会更多地把 CV 作为一种「技能」挂载。可证伪预测:未来两周 trending 上,带「官方/大厂背书」的 skills 仓库与协议类(MCP / AG-UI)仓库占比将维持高位,纯 prompt 合集继续退场。 Extrapolating from today's signals: (1) skill marketplaces standardize — with incumbents (Google, OpenAI) now participating directly, agent skills get discovered, versioned, signed, and audited like dependencies in a package manager; (2) dual protocols solidify — a lower MCP layer (tools/data) plus an upper AG-UI layer (frontend/interaction) likely co-define the standard stack for "agent apps," with CopilotKit's appearance an early signal; (3) retrieval & context keep getting cheaper — turbovec's quantized vector index charting again, combined with token compression, drives RAG unit cost further down; (4) perception won't vanish — supervision shows agents still need dedicated CV/multimodal tooling to "see the world," and future agent toolchains will increasingly mount CV as a "skill." Falsifiable prediction: over the next two weeks, skills repos with "official / incumbent backing" and protocol repos (MCP / AG-UI) stay a large share of trending, while pure prompt-collections keep fading.
