3. 趋势分析: 历史、当下、未来

3.1 历史: 从“模型服务化”到“状态服务化”

2022-2024 的 AI Infra 主线是把 Transformer 变成稳定 API:FlashAttention、PagedAttention、连续批处理、tensor parallel、token streaming、OpenAI-compatible endpoint。TGI、vLLM、TensorRT-LLM、llama.cpp、Ollama 分别从生产服务、吞吐、硬件优化、本地可移植和开发体验切入。这个阶段的核心指标是 tokens/sec、TTFT、显存占用、batch size、模型兼容性。

3.2 当下: Agentic workload 让瓶颈沿链路迁移

当下的核心变化是一次任务不再只是单次 chat completion。Agent 会读仓库、检索文档、调用工具、运行测试、写 PR、触发 CI、等待 review,再继续迭代。于是成本和可靠性瓶颈沿着“上下文 → prefill/KV → decode/kernel → router/scheduler → tools/agents → platform/governance”移动。

  • 证据 1: GitHub 5 月报告直接把流量增长归因于 AI-assisted / agentic workflows,并列出 Actions、Copilot code review、Copilot cloud agent、PR thread、Pages 等受影响面。
  • 证据 2: SGLang 0.5.14 的 DeepSeek-V4/GB300、MoE load balancing、NVFP4 和 Blackwell prefill kernel 说明 engine 优化已高度硬件/模型/工作负载定制。
  • 证据 3: NVIDIA Dynamo + llm-d 明确把 NIXL、Planner、KV Cache Manager 放进 prefill/decode disaggregation 和 KV tiering 的关键路径。
  • 证据 4: GAIE 将 Endpoint Picker 相关能力迁往 llm-d 生态,说明 Kubernetes-native inference 的标准边界正在从通用 gateway 细化到 inference-aware scheduler。

3.3 未来: AI Infra 会变成“分层 agent 操作系统”

未来 6-18 个月的趋势不是某一个 engine 通吃,而是栈分层更清楚:模型层选择能力和成本,推理层压吞吐和延迟,缓存层减少重复 prefill,协议层连接工具和数据,调度层管理 GPU 与 SLO,治理层处理权限、审计和复现。

  1. 模型层: frontier model 承接复杂推理,open-weight / local model 承接低延迟、隐私、成本敏感任务。
  2. 推理层: vLLM / SGLang / TensorRT-LLM / llama.cpp 分别服务云端高吞吐、复杂生成、NVIDIA 极致优化和边缘/本地可移植。
  3. 缓存层: KV cache 变成可配额、可压缩、可迁移、可观测的共享资源,并与 router/autoscaler 联动。
  4. 协议层: MCP 这类工具协议会像 HTTP/gRPC 之于微服务一样,成为 agent 接入外部世界的标准边界。
  5. 调度层: GAIE、llm-d、Kueue、Dynamo 会把 LLM 流量特征带进 Kubernetes 调度和多云容量规划。
  6. 治理层: prompt/response hash、tool-call audit、PII 处理、sandbox 隔离、policy enforcement 和 replay 会成为企业默认项。

4. 一句话结论

2026 年 6 月底的 AI Infra 关键词是 agentic pressure becomes systems pressure:agent 把推理成本、上下文复用、低精度硬件、工具协议、Kubernetes 调度、开发平台容量和合规审计同时推到前台。过去拼模型调用,现在拼系统闭环;未来拼的是能否把模型能力稳定、便宜、可恢复、可审计地接入真实工作流。
